# 레이아웃 선택 판단 (Decision Tree)

> 이 파일은 슬라이드 구성 설계 시 Read로 참조한다. SKILL.md에서 직접 로드하지 않음.

---

## 1단계: 콘텐츠 시그널 분석

사용자 콘텐츠에서 다음 시그널을 감지한다:

| 시그널 | 감지 패턴 | 1차 후보 |
|--------|-----------|----------|
| **서사/설명** | 문장형 텍스트, "~이란", "~하는 방법" | Text |
| **나열/항목 (2~3개)** | "첫째/둘째", 번호, 불릿, 2~3개 항목 | Card Grid 또는 Text(불릿) |
| **나열/항목 (4~6개)** | 기능, 특징, 서비스, 4~6개 항목 | Icon Grid / Bento Grid |
| **비교/대조** | "vs", "비교", "장단점", "before/after" | Two-Column / Pros/Cons |
| **다중 비교** | 3개+ 옵션 비교, 피처 매트릭스 | Comparison Matrix |
| **수치/트렌드** | 숫자 + 시간축, 증감, 비율, 추이 | Chart |
| **정형 데이터** | 표, 스펙, 스케줄 | Data Table |
| **핵심 지표** | KPI, 달성률, 목표치, 2~4개 핵심 숫자 | KPI |
| **시간순 데이터** | 연혁, 마일스톤, 날짜+이벤트 | Timeline |
| **단계별 과정** | 설치, 워크플로우, "1단계", "Step" | Step-by-Step |
| **전환률/파이프라인** | 퍼널, 전환, 깔때기 | Funnel |
| **순환 프로세스** | PDCA, 반복, 루프 | Cycle (SVG) |
| **분기별 계획** | Q1/Q2, 로드맵, 간트 | Roadmap |
| **겹침/교집합** | 벤 다이어그램, 공통점, 포지셔닝 | Venn (SVG) |
| **계층/우선순위** | 피라미드, 중요도, 프레임워크 | Pyramid |
| **코드/기술** | 코드 스니펫, API, 함수, 설정 | Code Block |
| **구조/흐름** | 아키텍처, 시스템 구성, 프로세스 | Image (SVG) |
| **인용/격언** | 인용문, 명언, 핵심 메시지 강조 | Quote |
| **전후 비교** | before/after, 이전/이후, 개선 결과 | Before/After |
| **목차/구조** | 목차, 아젠다, 순서 안내 | Agenda/TOC |
| **도입/마무리** | 제목, 인사, 감사, Q&A | Cover 또는 Text |
| **설치/환경구성** | 설치, 설정, 필수 조건, 버전 확인, npm install | Step-by-Step + Code Block |

> **통화 표기 규칙**: KPI·Chart에 달러($) 금액이 포함되면 원화 환산 병기. 예: `$2B (약 2.8조 원)`. `data-suffix`에 원화 추가 가능.

## 2단계: 항목수 기반 레이아웃 결정

나열형 콘텐츠의 항목 수로 레이아웃을 선택한다:

| 항목 수 | 레이아웃 | 이유 |
|---------|----------|------|
| 2개 | Two-Column / Before/After / Pros/Cons | 좌우 비교 최적 |
| 3개 | Card Grid (1행) / Pyramid (계층 시) / Cycle (순환 시) | 3열 그리드에 딱 맞음 |
| 4개 | KPI (수치) / Icon Grid / Bento Grid | 4열 KPI 최적, 기능 나열은 Icon Grid |
| 5개 | Text (불릿) / Step-by-Step (단계 시) / Funnel (단계+전환 시) | 단계형이면 Step/Funnel, 아니면 불릿 |
| 6개 | Card Grid (2행) / Icon Grid / Chart (6 bar) | 데이터면 Chart, 기능은 Icon Grid, 일반은 Card |
| 4~6개 (분기별) | Roadmap / Timeline | 시간축 있으면 Timeline/Roadmap |
| 3~5개 (비교) | Comparison Matrix | 다중 옵션 비교표 |
| 7~12개 | Chart (bar) / Data Table / 슬라이드 분할 | 정형이면 Table, 시각화면 Chart |
| 13개+ | 반드시 분할 | 단일 슬라이드 불가 |

## 3단계: 슬라이드 흐름 패턴

전체 프레젠테이션의 슬라이드 시퀀스:

```
[Cover] → [Agenda] → [Section Divider] → [본론 Mix] → [데이터] → [Quote/마무리]
```

**본론 Mix 원칙:**
- 같은 레이아웃 2장 연속까지만 (3장 연속 금지)
- 시각적 밀도 교대: 정보 밀집(Table/Matrix) → 여백(Quote/Divider) → 중간(Card/Icon)
- SVG 다이어그램(Cycle/Venn)은 앞뒤에 Text/Card로 맥락 제공
- 데이터 슬라이드(Chart/KPI)는 설명 슬라이드 뒤에 배치 (맥락 → 증거)
- Code Block은 앞뒤에 Text 슬라이드로 맥락 제공
- Section Divider로 대주제 전환을 명확히 구분

## 4단계: 콘텐츠 부족 시 보강

사용자가 제목만 주거나 콘텐츠가 빈약한 경우:
- **Text 슬라이드**: 주제에서 핵심 포인트 3~5개 도출
- **Card Grid**: 주제의 하위 카테고리/특징을 3개로 구조화
- **Chart**: 샘플 데이터로 트렌드 패턴 생성 (사용자에게 "실제 데이터로 교체하세요" 안내)
- **KPI**: 대표 지표 4개를 placeholder로 생성
- **Timeline**: 대표 마일스톤 3~5개를 연도+이벤트 형태로 생성
- **Step-by-Step**: 주제의 핵심 단계를 3~5개로 구조화
- **Funnel**: 일반적인 전환 단계(인지→관심→전환→유지)로 생성
- **Comparison Matrix**: 주요 옵션 3개 × 핵심 기준 4개로 placeholder 생성
- **Data Table**: 헤더 + 샘플 4~6행으로 구조화 (사용자에게 "실제 데이터로 교체하세요" 안내)
- **Icon Grid**: 주제 관련 기능/특징 4~6개를 아이콘+라벨로 구조화
- **Bento Grid**: 핵심 1개(wide) + 보조 2~3개로 비대칭 배치
- **Quote**: 주제 관련 대표 인용구 또는 핵심 메시지 1문장 생성
- **Section Divider**: 대주제 전환 시 번호 + 제목으로 구성
- **Roadmap**: Q1~Q4 또는 Phase 1~3으로 placeholder 생성
- **Before/After**: 이전/이후 상태를 대비하여 구조화
- **Pros/Cons**: 장점 3개 + 단점 3개로 placeholder 생성
- **Pyramid**: 3~4단 계층으로 구조화 (상위 = 핵심, 하위 = 기반)
- **Agenda/TOC**: 프레젠테이션 전체 섹션을 번호 목록으로 구성
- **Venn Diagram**: 2~3개 영역의 공통점/차이점으로 구조화
- **Cycle**: 3~5단계 순환 과정으로 구조화

## 분량 초과 판단 규칙

콘텐츠가 단일 슬라이드 상한을 초과하면 **무조건 분할**:

1. **텍스트 분할**: 논리적 단위로 나눔 (주제별, 단계별)
2. **카드 분할**: 3개씩 끊어서 Part 1, Part 2
3. **데이터 분할**: 카테고리별로 Chart/KPI 분리
4. **코드 분할**: 함수/모듈 단위로 분리, 각 슬라이드에 맥락 주석 추가
5. **테이블 분할**: 행이 6개 초과 시 카테고리별로 분리
6. **Timeline 분할**: 이벤트 6개 초과 시 기간별로 분리
